# 2026-05-14 更新记录

## 背景

- 将现有纯文字“使用指南”改为页面内交互式引导，参考夜雀助手的高亮步骤式教程体验。
- 解决首次进入时用户直接关闭教程、跳过关键说明的问题。
- 为后续自动化调试补充可控的 debug 参数，减少反复被首屏教程和真实存档依赖打断。

## 变更范围

- 交互式教程控制器：`src/components/AppGuide.tsx`
  - 改为基于 `driver.js` 的页面高亮式教程。
  - 首次自动弹出默认要求完整阅读。
  - 手动点击顶部“使用指南”时，教程从一开始就允许跳过。
  - 导入配置成功后，会立即解锁当前教程的“跳过”按钮。
  - 设置页新增“先导入存档或配置才能继续”的强制门槛；如果到第 6 步仍拿不到文件，可直接在教程里导入临时假存档继续体验。
  - 第 7 步和第 9 步现在会自动替用户切到“妖怪兽道”，不再要求手动打开地区下拉框。
- 教程状态持久化：`src/stores/game-store.ts`
  - 将旧的“是否自动弹出”升级为“教程完成版本 + 是否解锁跳过”。
  - 配置导入时自动将用户视为已有使用经验，解锁跳过能力。
  - 兼容旧的 `guideAutoOpenDisabled` 持久化字段。
  - 新增 `guideDataSource`，区分真实存档、配置导入、临时假存档与未导入状态，供教程强制门槛使用。
- 调试模式：`src/lib/debug-mode.ts`、`src/App.tsx`
  - `?no_tutorial=true`：关闭首次自动弹出教程。
  - `?fake_save=true`：自动注入一份假配置并默认选中 `人间之里`，便于调试页面。
  - 为避免污染真实本地状态，debug 假配置会在退出该参数后自动恢复原存档缓存。
- 页面锚点与文案：`src/pages/SettingsPage.tsx`、`src/pages/NormalPage.tsx`、`src/pages/RarePage.tsx`、`src/index.css`
  - 为设置、普客、稀客页面补充稳定的教程锚点。
  - 设置页顶部新增“先导入才能继续”的提示，以及临时假存档状态提示。
  - 稀客页在未选中目标稀客时新增占位提示，明确告知需要先选人才能继续配置点单 Tag。
  - 新增教程气泡样式，统一为夜雀掌柜当前视觉风格。
  - 修复教程过程中 Base UI Select 与 `driver.js` 同时启用时的指针事件冲突，恢复第 7 步与第 9 步地区下拉框的正常点击。
  - 调整第 11 步的教程锚点与气泡位置，避免遮挡“点单酒水 Tag”按钮，确保 10-13 步可以完整手动走通。
- 文档同步：`README.md`、`.github/copilot-instructions.md`
  - README 补充交互式教程与“无本地存档时走配置导入或手动设置”的路径说明。
  - 协议文件补充“任务完成后必须先提问用户下一步需求”的收尾要求。

## 验证结果

- `pnpm lint`：通过。
- `pnpm build`：通过。
- 浏览器验收：通过。
  - 第 6 步在未导入时会锁住“下一步”，并显示“导入临时假存档”按钮。
  - 选择临时假存档后，会写入全部食谱、全部酒水、全部食材，食材库存为 100。
  - 第 7 步和第 9 步会自动切到“妖怪兽道”，并直接开放“下一步”。
  - 第 11 步不会再遮挡点单 Tag 操作区，10-13 步可连续完成并写入教程完成状态。
- 构建产物存在 chunk size warning，但本次改动未导致构建失败。

## 风险与备注

- 当前教程以“高亮步骤 + 手动下一步”为主，没有把所有 UI 操作都做成严格的状态机式自动推进，后续如需更贴近夜雀助手的“必须点击指定控件才前进”，可以继续加细。
- `?fake_save=true` 主要面向调试，不建议作为普通用户的长期使用入口。
