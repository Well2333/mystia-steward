# 2026-04-30 更新记录

## 背景

- 在真实存档里，料理流行 tag 的数字 id 映射此前被临时禁用，以避免把错误编号解析成错误标签。
- 本轮基于用户提供的完整游戏文件，直接从游戏 Addressables 资源中恢复了权威的 food tag id 映射，目标是安全恢复自动识别功能。

## 变更范围

- 通过 `catalog.bundle` 定位 `FoodTagProfile.asset`、`FoodTagRuleProfile.asset` 与中文 `FoodTagsLang.txt` 的真实 Addressables 入口。
- 确认内容 bundle 采用固定 `0x53` 异或后即可还原为标准 UnityFS 包，并据此成功解出目标资源。
- 从 `FoodTagProfile.asset` 的 `indexes` 与中文 `FoodTagsLang.txt` 恢复权威的 food tag id -> 名称映射。
- 新增 `src/data/food-tag-id-map.json`，把已核实的 tag id 映射沉淀为结构化数据。
- 更新 `src/lib/save-parser.ts`，恢复对料理流行数字 tag id 的自动解析，并保留字符串值兼容路径。
- 更新 `src/lib/save-parser.ts`，接入真实存档里的 `dayScenePartial.trackedSwitch.Aya_FamousIzakaya`，将其导入为 `famousShopEnabled=true`。
- 调整 `src/lib/save-parser.ts` 的导入语义：当 `dayScenePartial.trackedSwitch.Aya_FamousIzakaya=true` 且流行喜爱 tag 为 `招牌` 时，不再把 `招牌`写入 `popularFoodTag`，而是转为仅开启明星店效果。
- 更新 `src/pages/SettingsPage.tsx`，让“导入成功! 料理: x, 酒水: x”提示额外显示本次导入后的流行料理标签状态（喜爱、厌恶、明星店）。
- 更新 `src/lib/food-tags.ts`，补齐当前仓库可直接消费的动态料理标签：`实惠`、`昂贵`、`大份`。
- 新增 `docs/addressables-tag-mapping-playbook.md`，沉淀从 Addressables 反查 tag id 映射的完整流程。
- 更新 `docs/tmi-cooking-mechanics-knowledge-base.md`，将“数字映射已禁用”的旧说明改为已核实、已恢复。

- `煮锅`、`烧烤架`、`油锅`、`蒸锅`、`料理台`、`黑暗物质`、`厨具「夜雀」` 等系统型 tag 虽然已完成 id 核实，但本仓库当前并未把它们纳入流行料理设置项或推荐消费路径，因此导入时继续忽略。
