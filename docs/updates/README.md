# 更新记录规范

本目录用于记录任意更新的时效性日志（代码、数据、配置、文档、验收等）。

## 记录范围

- 任何需要追溯的当次更新都应写入本目录。
- 上游同步只是其中一种场景，不是唯一场景。

## 命名建议

- `<日期>-update.md`（例如 `2026-04-23-update.md`）
- 同日多次更新可追加后缀：`2026-04-23-update-2.md`

## 内容模板

```markdown
# YYYY-MM-DD 更新记录

## 背景
- 本次更新目的

## 变更范围
- 文件列表
- 关键行为变化

## 验证结果
- lint/build
- 业务脚本校验
- 浏览器验收（如有）

## 风险与备注
- 未解决项
- 后续动作
```

## 注意事项

- 本目录存放的是时效日志，不是长期规则。
- 长期规则应写在对应规则文档（例如 development-conventions、repo-memory、专项手册）中。
- 禁止把时效日志写回 `docs/repo-memory.md` 或流程手册（例如 `docs/upstream-sync-playbook.md`）。
