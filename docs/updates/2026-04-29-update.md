# 2026-04-29 更新记录

## 背景

- 接入存档导入后的自动设置映射，让现有单标签流行模型与联动稀客显隐能直接消费存档信息。
- 本轮范围限定为：料理流行标签、`collabStatus`；酒水流行与 `rewindMode` 只解析，不接入现有推荐与 UI。

## 变更范围

- 扩展 `src/lib/save-parser.ts`：新增对 `popLikeFoodTags`、`popHateFoodTags`、`popLikeBevTags`、`popHateBevTags`、`collabStatus`、`rewindMode` 的防御式解析。
- 修正真实存档字段路径：上述字段优先从 `playerPartial` 读取，并兼容回退到 `storagePartial`。
- 修正真实存档料理流行值格式：`popLikeFoodTags` / `popHateFoodTags` 支持数字 tag id 映射到现有料理标签。
- 修正导入互斥规则：若存档导入出流行喜好标签，则会显式清空流行厌恶，避免出现二者同时显示。
- 新增 `src/lib/food-tags.ts`，统一可导入的料理标签集合，并复用到设置页与存档解析。
- 新增 `src/lib/special-customer-visibility.ts`，集中管理联动角色与 DLC9 角色的默认隐藏与按存档解锁逻辑。
- 更新 `src/stores/game-store.ts` 的 `importSave`：
  - 料理流行数组按“第一个有效值”写入 `popularFoodTag` 与 `popularHateFoodTag`
  - 流行喜好存在时，`popularHateFoodTag` 强制清空
  - 联动角色与 DLC9 角色默认隐藏
  - `collabStatus` 与 DLC 激活状态只驱动 `rareHiddenCustomerIds` 的解除隐藏
  - 保留非特殊来源的手动 `rareExtraCustomerIds`
- 已实现联动映射：`MC_Gensokyo -> [30]`、`3FARIES_Collab -> [31]`、`TBS_Kokoro -> [41]`、`TBC2_Collab -> [36, 37, 38]`
- `THYG` 作为合法空映射处理，不产生角色变化；未知 `collabStatus` key 继续忽略。
- 更新 `docs/tmi-cooking-mechanics-knowledge-base.md`，补充本轮导入语义与边界。
- 更新 `.playwright-cli/playwright-advanced-acceptance.fn.template.js`，改用 `Mystia#17.memory` 并显式断言导入后的热门料理标签互斥与特殊稀客展示。
- 修正设置页空值展示：流行料理标签空值从内部哨兵 `_none` 改为用户可读的“无”，并同步调整 Playwright 验收断言。
- 移除 `scripts/verify-rare-ingredient-availability.ts`，并从开发约定与同步流程中删除该校验步骤。
- 增强 `.playwright-cli/playwright-advanced-acceptance.fn.template.js`：新增非法配置导入提示、利润开关跨页生效、特殊稀客负向显隐、添加/移除稀客、稀客收藏与过滤放宽等更贴近真实操作的断言。
