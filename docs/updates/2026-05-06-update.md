# 2026-05-06 更新记录

## 背景

- 基于 issue #1 的核查结论，落地本项目确认要修复的稀客推荐问题：
  - 彩蛋规则缺失
  - 流行趋势与互斥覆盖链路残留
  - 黑暗物质在梅蒂欣场景下的展示策略

## 变更范围

- 推荐算法：`src/lib/rare-recommend.ts`
  - 新增稀客彩蛋规则表，支持“彩蛋优先展示（ExGood）”与“禁推”两类效果。
  - 接入规则（当前版本）：
    - 优先展示：蕾米莉亚×猩红恶魔蛋糕、饕餮尤魔×油豆腐、雾雨魔理沙×牛肉鸳鸯火锅、河城荷取×黄瓜。
    - 禁推：古明地恋×无意识妖怪慕斯、绵月丰姬/依姬×蜜桃红烧肉、犬走椛×可可豆。
  - 增加彩蛋相关食材候选约束：
    - 禁推食材不进入候选。
    - 彩蛋优先食材可强制进入候选。
  - 修复流行趋势链路：改为“先互斥覆盖，再基于有效标签判定流行喜爱/厌恶”。
  - 最终排序增加彩蛋优先级，彩蛋优先结果固定置顶。

- 类型扩展：`src/lib/types.ts`
  - `IRareRecipeResult` 新增：
    - `isEasterPriority`
    - `easterReason`

- 稀客页面：`src/pages/RarePage.tsx`
  - 料理列表前端排序与算法优先级对齐：彩蛋优先结果固定置顶。
  - 彩蛋优先料理卡片新增“彩虹边框 + 彩蛋优先”标识。
  - 在梅蒂欣信息区域新增备注：黑暗物质不纳入常规料理推荐。

- 机制文档同步：`docs/tmi-cooking-mechanics-knowledge-base.md`
  - 更新动态标签处理口径（互斥后再判定流行）。
  - 增补彩蛋规则策略与当前接入清单。
  - 补充黑暗物质“仅备注、不并入常规推荐”说明。
