# 2026-05-06 更新记录（2）

## 背景

- 根据需求继续细化彩蛋逻辑口径与 UI 行为：
  - 正面彩蛋仅提供得分下限，不再无条件改判 ExGood。
  - 河城荷取 + 黄瓜改为“加料食材高亮”而非料理整卡高亮。
  - 彩蛋说明迁移至角色卡片区域。
  - 新增“彩蛋料理特殊显示”设置开关，默认关闭。

## 变更范围

- 推荐算法：`src/lib/rare-recommend.ts`
  - 正面彩蛋规则新增 `scoreFloor`，用于设置料理得分下限。
  - 取消“正面彩蛋无条件改判 ExGood”的处理，改为：
    - 先应用得分下限
    - 再走通用评级函数
    - 仍需满足点单料理 tag 才能得到 ExGood
  - 彩蛋规则拆分为：是否置顶（`pinOnQualified`）、是否料理卡高亮（`recipeHighlight`）、是否加料食材高亮（`ingredientHighlightIds`）。
  - 河城荷取 + 黄瓜规则设定为：`pinOnQualified=false`、`recipeHighlight=false`，仅高亮指定加料食材。

- 稀客页排序：`src/pages/RarePage.tsx`
  - 调整为条件置顶分组排序：星标+彩蛋 > 星标 > 彩蛋 > 其他（其中彩蛋置顶需满足点单料理 tag，且需开启显示开关）。
  - 彩蛋说明从料理区迁移至角色卡片区域。
  - 料理卡片仅在规则允许时应用彩虹边框；加料食材支持独立彩虹边框（用于河城荷取 + 黄瓜）。
  - 彩蛋说明文案改为固定句式：`{角色}对{料理或食材}的评价为{评价内容}。`
  - 梅蒂欣角色卡说明同步改为固定句式：梅蒂欣对黑暗物质的评价为3分。

- 当前彩蛋说明文案（用户侧）
  - 蕾米莉亚对猩红恶魔蛋糕的评价为4分。
  - 饕餮尤魔对油豆腐的评价为3分。
  - 雾雨魔理沙对牛肉鸳鸯火锅的评价为4分。
  - 河城荷取对黄瓜的评价为3分。
  - 古明地恋对无意识妖怪慕斯的评价为极差。
  - 绵月丰姬对蜜桃红烧肉的评价为极差。
  - 绵月依姬对蜜桃红烧肉的评价为极差。
  - 犬走椛对可可豆的评价为极差。

- 设置与持久化：`src/stores/game-store.ts`、`src/pages/SettingsPage.tsx`、`src/lib/config-io.ts`
  - 新增全局持久化开关 `rareEasterVisualEnabled`，默认 `false`。
  - 设置页新增“彩蛋料理特殊显示”开关。
  - 配置导入导出升级到 `v7`，新增 `ev` 字段并保持旧版本兼容（缺失字段按 `false` 处理）。

- 机制文档：`docs/tmi-cooking-mechanics-knowledge-base.md`
  - 同步更新为“得分下限 + 仍需点单 tag 约束 + 开关门控 + 河城荷取食材级高亮”。
