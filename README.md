# 夜雀掌柜

**夜雀掌柜 Mystia Steward** 是一个本地前端工具，用来辅助你在《东方夜雀食堂》中：

- 给普客快速排出“更值得上桌”的料理与酒水
- 给稀客按点单标签计算“完美优先”的料理 + 加料 + 酒水
- 管理可用料理/酒水/食材，并保存配置

它不改游戏存档，只读取你导入的数据并在浏览器中计算推荐结果。

---

## 功能总览

### 1) 设置页（`/settings`）

- 导入存档（支持拖拽 `.memory` 文件）
- 料理/酒水/食材三态过滤：
  - `全可用`：普客 + 稀客都可用
  - `仅稀客`：只在稀客推荐中使用
  - `禁用`：不参与推荐
- 取得状态编辑（已取得/未取得）
- 食材库存阈值批量设置（低于 X 个）
- 流行标签设置（喜爱 / 厌恶）
- 配置导入导出（剪贴板 / 文件）

### 2) 普客页（`/normal`）

- 按地区查看普客需求
- 料理二级筛选：全部 / 按厨具（煮锅、烧烤架、料理台、油锅、蒸锅）
- 料理排序：覆盖优先 / 利润优先
- 酒水排序：匹配优先 / 高价优先
- 侧边栏显示地区普客标签概览

### 3) 稀客页（`/rare`）

- 地区内稀客快速切换，支持隐藏未遇到稀客
- 点单料理 Tag / 点单酒水 Tag 选择并持久化
- 料理结果支持：
  - 筛选模式切换：
    - 隐藏非完美（默认）
    - 隐藏低于 X 分以下料理（X 可选 0-3）
  - 厨具、基础原料（绿底）、加料（黄底 `+`）展示
  - 加料会标注用途标签（如 `+葡萄(甜,果味)`）
  - 悬浮 Tag 明细（命中喜好粉色、命中厌恶红色、相消标签棕色删除线）
- 酒水结果按“满足点单优先 + 价格降序”

## 使用教程

- 首次进入时，会自动弹出交互式新手引导，按“设置 → 普客 → 稀客”带你走一遍核心流程。
- 首次自动弹出默认需要完整阅读；如果你是手动点击顶部“使用指南”打开，或已经导入过配置文件，则可以直接跳过。
- 引导在设置页会强制要求你先导入存档或配置；如果当前设备确实拿不到文件，引导第 6 步可以临时导入一份演示假存档继续体验。
- 顶部“使用指南”按钮始终可用，后续可随时重新进入教程。

### 导入存档

进入 `设置` 页：

1. 点击“选择存档文件”或直接拖拽 `Mystia#x.memory`
2. 导入成功后，已取得的料理/酒水/食材会自动写入状态，页面会同时显示本次导入后的流行料理标签结果：喜爱、厌恶，以及“明星店”是否开启
3. 如需手动修正，打开“调整取得状态”开关逐项点选

如果当前设备没有本地存档：

1. 直接使用“配置导入 / 导出”区域导入之前备份的配置字符串或配置文件
2. 如果也没有配置文件，可以先在有存档的设备上导入夜雀掌柜，再把导出的配置字符串或配置文件带到当前设备
3. 如果引导过程中仍然拿不到文件，可临时导入一份演示假存档：它会开放全部食谱、全部酒水、全部食材，并把食材库存填成 100，方便继续体验流程
4. 如果你不想用临时假存档，再打开“调整取得状态”手动勾选已取得项目

说明：

- 若真实存档中存在 `dayScenePartial.trackedSwitch.Aya_FamousIzakaya=true`，且流行喜爱标签为 `招牌`，导入结果会显示为“喜爱=无，明星店=开启”，因为 `招牌` 会转为明星店效果而不占用流行喜爱槽位
- 夜雀食堂的存档机制有时会导致“明星店”识别不够稳定；即使系统自动解析了它，也建议你在设置页手动确认它当前是否仍然生效，或其实已经过期

### 设置过滤策略

在设置页的三个标签中分别调整：

- 料理：设为全可用/仅稀客/禁用
- 酒水：设为全可用/仅稀客/禁用
- 食材：设为全可用/仅稀客/禁用，并可用“低于 X 个食材”做批量策略

建议：

- 普客常用料理保持 `全可用`
- 高成本/稀有素材可以设为 `仅稀客`
- 不想出现的内容设为 `禁用`

### 查看普客推荐

进入 `普客` 页：

1. 选择地区
2. 切换“料理/酒水”标签
3. 在“料理”标签下可按厨具切换（全部/煮锅/烧烤架/料理台/油锅/蒸锅）
4. 根据你当前目标切换排序：
   - 冲收益：利润优先
   - 冲覆盖：覆盖优先

### 查看稀客推荐

进入 `稀客` 页：

1. 选地区后点击目标稀客头像
2. 选择“点单料理 Tag”和“点单酒水 Tag”
3. 在稀客页筛选栏切换筛选模式（默认“隐藏非完美”）
4. 若切换模式，系统会弹窗说明两种模式区别
5. 鼠标悬浮料理可看 tag 匹配明细

## 配置导入/导出（跨设备）

设置页提供两种方式：

- 导出到剪贴板（压缩字符串）
- 导出为文件（默认 `mystia-steward-config.txt`）

导入时支持：

- 粘贴配置字符串导入
- 选择配置文件导入

适合在多台设备间同步过滤配置和偏好设置。

## 评分与规则说明

- 逻辑层评价等级：`ExGood` / `Good` / `Normal` / `Bad` / `ExBad`
- 页面展示会按 `粉 / 橙 / 绿 / 紫 / 黑` 颜色映射这些等级
- 命中顾客厌恶 tag 仅扣分（`-1`），不是一票否决
- 互斥标签会按规则抵消弱方（如 肉 > 素）
- 仍会保留禁忌过滤（会导致黑暗物质的组合不纳入推荐）
